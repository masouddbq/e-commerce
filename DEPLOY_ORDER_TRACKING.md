# راهنمای Deploy تغییرات - انتقال پیگیری سفارش به پنل کاربری

## 📋 خلاصه تغییرات

بخش پیگیری سفارش از صفحه اصلی (لندینگ) به پنل کاربری منتقل شد. حالا کاربران باید وارد حساب کاربری شوند تا بتوانند سفارش خود را پیگیری کنند.

---

## 🚀 مراحل Deploy

### گزینه 1: Build روی سیستم محلی (توصیه می‌شود)

#### مرحله 1: Build کردن Frontend

```powershell
# در مسیر پروژه (روی سیستم خودتان)
cd e:\FrontProjects\Lent-shop-new\lent-shop

# نصب dependencies (اگر تغییر کرده‌اند)
npm install

# Build کردن پروژه
npm run build
```

**نکته:** بعد از build، فایل‌های آماده در پوشه `dist/` قرار می‌گیرند.

#### مرحله 2: انتقال فایل‌های Build شده به سرور

**روش 1: استفاده از FTP/FileZilla**
1. به سرور خود متصل شوید
2. محتوای پوشه `dist/` را کپی کنید
3. در مسیر هاست (معمولاً `C:\inetpub\wwwroot\lent-shop\`) جایگزین کنید

**روش 2: استفاده از Remote Desktop**
1. به سرور متصل شوید (RDP)
2. فایل‌های `dist/` را از سیستم محلی به سرور کپی کنید
3. در مسیر هاست جایگزین کنید

**روش 3: استفاده از PowerShell (اگر دسترسی دارید)**
```powershell
# روی سرور
xcopy "E:\FrontProjects\Lent-shop-new\lent-shop\dist\*" "C:\inetpub\wwwroot\lent-shop\" /E /Y
```

---

### گزینه 2: Build روی سرور (اگر Node.js روی سرور نصب است)

#### مرحله 1: انتقال فایل‌های تغییر یافته به سرور

فایل‌های زیر را به سرور منتقل کنید:
```
src/
├── Components/
│   ├── HomePage/
│   │   └── HomePage.jsx          ✅ تغییر یافته (حذف OrderTracking)
│   └── Pages/
│       └── UserAccount.jsx       ✅ تغییر یافته (اضافه شدن OrderTracking)
```

#### مرحله 2: Build روی سرور

```powershell
# روی سرور - در مسیر پروژه
cd C:\path\to\lent-shop

# نصب dependencies (اگر تغییر کرده‌اند)
npm install

# Build کردن
npm run build
```

#### مرحله 3: کپی فایل‌های Build شده به هاست

```powershell
# کپی فایل‌های dist به هاست IIS
xcopy "C:\path\to\lent-shop\dist\*" "C:\inetpub\wwwroot\lent-shop\" /E /Y
```

## ✅ چک‌لیست Deploy

### قبل از Deploy:
- [ ] فایل‌های تغییر یافته را بررسی کرده‌اید
- [ ] Build بدون خطا انجام شده است
- [ ] فایل `dist/index.html` وجود دارد
- [ ] فایل `dist/web.config` وجود دارد

### بعد از Deploy:
- [ ] صفحه اصلی (لندینگ) بدون بخش پیگیری سفارش باز می‌شود
- [ ] وارد حساب کاربری می‌شوید (`/account`)
- [ ] بخش "پیگیری سفارش" در پنل کاربری نمایش داده می‌شود
- [ ] می‌توانید با شماره سفارش یا کد پیگیری جستجو کنید
- [ ] جزئیات سفارش به درستی نمایش داده می‌شود

---

## 🔍 تست کردن

### 1. تست صفحه اصلی:
```
https://your-domain.com/
```
- بررسی کنید که بخش پیگیری سفارش دیگر نمایش داده نمی‌شود

### 2. تست پنل کاربری:
```
https://your-domain.com/account
```
- وارد حساب کاربری شوید
- بخش "پیگیری سفارش" را در بالای صفحه ببینید
- یک شماره سفارش یا کد پیگیری را وارد کنید
- بررسی کنید که جزئیات سفارش نمایش داده می‌شود

## 📝 نکات مهم

1. **Backend Server:** نیازی به تغییر در سرور Express نیست (فقط تغییرات Frontend)

2. **Cache:** بعد از deploy، ممکن است نیاز به پاک کردن cache مرورگر باشد:
   - `Ctrl + Shift + R` (Windows)
   - `Cmd + Shift + R` (Mac)

3. **IIS Restart:** معمولاً نیازی به restart IIS نیست، اما اگر تغییرات اعمال نشد:
   ```powershell
   iisreset
   ```

4. **Backup:** قبل از deploy، از فایل‌های قبلی backup بگیرید

---

## 🆘 عیب‌یابی

### مشکل: تغییرات اعمال نشده
- Cache مرورگر را پاک کنید
- بررسی کنید که فایل‌های `dist/` به درستی کپی شده‌اند
- IIS را restart کنید

### مشکل: خطا در Build
```powershell
# پاک کردن node_modules و نصب مجدد
rm -r node_modules
npm install
npm run build
```

### مشکل: بخش پیگیری سفارش نمایش داده نمی‌شود
- بررسی کنید که فایل `dist/index.html` به‌روز است
- بررسی کنید که فایل‌های JavaScript در `dist/assets/` وجود دارند
- Console مرورگر را بررسی کنید (F12)
